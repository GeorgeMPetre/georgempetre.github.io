1. Test Plan Identifier

OpenCart_XAMPP_UI_API_Performance_v1.1

2. Introduction

This Test Plan outlines the strategy, scope, and execution approach for validating the functional and non-functional quality of the OpenCart e-commerce application hosted locally on XAMPP. Testing includes UI functional testing, REST API validation, and comprehensive performance engineering such as load, stress, scalability, endurance, and volume testing using Apache JMeter.

3. Test Items

OpenCart Frontend (Customer Storefront), OpenCart Backend (Admin Portal), REST API Endpoints (Authentication, Products, Cart, Orders), and the server/infrastructure under performance evaluation.

4. Features to be Tested

UI Functional Testing

User login and registration, product browsing and filters, cart operations (add, edit, remove), checkout and order confirmation.

Admin Testing

Admin login, product CRUD operations, order management.

API Testing

Login and token generation, product list retrieval, cart add/remove actions, order submission.

Performance Testing

Load testing (10–100–250–500 users), stress testing, scalability testing, endurance testing, volume testing, throughput and latency analysis, and resource monitoring via PerfMon (CPU, memory, disk, network).

5. Features Not to be Tested

Payment gateways, third-party modules, mobile/responsive layout, email or SMS notifications, mobile emulation, and any external integrations outside core OpenCart.

6. Approach

Manual Functional Testing

Exploratory and scenario-based testing on Chrome, Firefox, and Edge.

Automation Testing

PyTest with Page Object Model, soft assertions with screenshots, HTML and JSON reporting, parallel execution where applicable.

API Testing

Postman collections for modelling flows, automated API validation using Python requests, covering positive, negative, and security scenarios.

Performance Testing

Apache JMeter 5.6.3 with CSV-driven data, Uniform Random and Constant Throughput timers, AutoStop thresholds (>3s ART or >1% errors), and PerfMon-based resource monitoring.

7. Item Pass/Fail Criteria

Functional Criteria

Pass: All high-priority tests pass with no critical issues. Fail: Any defect causing data loss, security issues, or repeated critical failures.

Performance Criteria (SLA)

95% of requests under 3 seconds, error rate ≤1%, and system stability without resource exhaustion. Tests fail if thresholds are exceeded or the application becomes unresponsive.

8. Suspension and Resumption Requirements

Testing is suspended in cases of server crashes, persistent 5xx errors, database lockups, unstable environment, or blocked critical functionality. Testing resumes once issues are fixed, the environment is stabilised, and required resets are completed.

9. Test Deliverables

Test Plan, functional test cases, automated PyTest suite, API Postman collection, JMeter test plans, performance reports, traceability matrix, defect logs, and final test summary report.

10. Testing Tasks

Scope review, test case design, automation development, API modelling, execution of UI/API/Performance tests, load and stress modelling, defect reporting and retesting, final reporting, and portfolio publication.

11. Environmental Needs

Software

Windows 11, XAMPP (Apache, MySQL, PHP), Chrome, Firefox, Edge, PyCharm, Postman, JMeter, Selenium, Python.

Optional Monitoring Tools

JMeter PerfMon plugin, Windows Resource Monitor, Apache access/error logs, MySQL slow query logs.

13. Schedule

Week 1 – Scope & Test Plan creation. Week 2 – Manual Test Case development. Week 3 – Automation & API testing. Week 4 – Load, Stress, Scalability testing. Week 5 – Reporting & portfolio updates.

14. Risks & Contingencies

Risks include XAMPP environment instability, hardware limitations, lack of production-like datasets, and documentation gaps. Mitigation includes gradual load increases, AutoStop rules, frequent environment resets, dataset generation, and validation using logs and PerfMon metrics.
